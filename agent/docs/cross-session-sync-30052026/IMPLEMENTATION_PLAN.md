## Ripple Effects

| Change | Risk | Mitigation |
|--------|------|------------|
| Adding code to BOTH data handlers (terminal:create + spawn-terminal) | Medium — duplicate code, one could be missed | Extract shared logic into a function, call from both |
| `parseAndExecuteActions` batch event change | Low — adding fields to existing event, backward compatible | Old consumers ignore new fields |
| Lock cleanup in terminal kill | Low — existing terminals unaffected, locks expire anyway | Test by killing terminal with active lock |
| New preload bridges | Low — new methods, no existing code changes | Verify each bridge returns expected shape |
| Type declarations | Low — adding new optional methods | Old code doesn't call them |
| `onContextChanged` re-enabled | Medium — was removed before for unknown reasons | Keep logic additive (refreshes + broadcasts), don't break existing flow |
| `/sync` interception | Low — only triggers on strings starting with `/sync` | Normal instructions unaffected |
