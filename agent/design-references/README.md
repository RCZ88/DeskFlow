# Design References Library

Curated DESIGN.md files from production sites that match DeskFlow's aesthetic: dark theme, glassmorphism, data-heavy dashboards, developer tools.

## Available References

| Reference | File | Style | Best For |
|-----------|------|-------|----------|
| **Claude** | `claude/DESIGN.md` | Warm terracotta, clean editorial | AI agent interfaces, chat UIs |
| **Linear** | `linear/DESIGN.md` | Ultra-minimal, purple accent | Project management, task lists |
| **Vercel** | `vercel/DESIGN.md` | Black/white precision, Geist | Dashboards, developer tools |
| **Stripe** | `stripe/DESIGN.md` | Purple gradients, weight-300 | Financial data, tables, charts |
| **Supabase** | `supabase/DESIGN.md` | Dark emerald, code-first | Developer tools, API docs |
| **Sentry** | `sentry/DESIGN.md` | Dark dashboard, data-dense | Error tracking, monitoring |
| **PostHog** | `posthog/DESIGN.md` | Playful dark, colorful charts | Analytics, funnels, graphs |
| **Raycast** | `raycast/DESIGN.md` | Sleek dark chrome, vibrant | Command palettes, quick actions |

## Usage

In the agent workspace, enable "Design References" in the Design Skills section of the Setup dialog. The agent can then reference these files with commands like:

- "Apply Linear style to this task list"
- "Use Vercel's dashboard pattern for the stats page"
- "Follow Stripe's data table conventions"

## Format

All DESIGN.md files follow the Stitch format:
- `## Design Tokens` — Colors, typography, spacing
- `## Components` — Button, Card, Input, etc.
- `## Patterns` — Layouts, navigation, feedback
- `## Examples` — Screenshots described as text
